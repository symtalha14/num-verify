Num-Verify

A simple phone number testing API

What is Num-Verify?

Num-Verify is an Open Source , Free to Use API which can be used to verify phone numbers across the World according to their Prefixes. It is written in Django 2.2 on Python 3.6 . It is available for both Hosted and Installation based use in your Application. You can also fork the same on GitHub and make your own contributions.

Request Parameters

Use the following parameters to make a POST request to the API


Parameter Significance Type
NUM Specify the phone number that you want to verify. String or Integer
NUM_PRE Specify the prefix to which the number is associated. String or Integer
Note: The parameter names are Case-Sensitive

Routes

GET Request to the API would fetch all the country codes data in JSON format :


GET: api/fetch/all


POST Request can be made to the following Route :


POST: api/fetch/

NOTE: GET request to this POST route is not allowed

Media types allowed:


Media-Type Content-Type (Header)
JSON application/json
x-www-form-urlencoded application/x-www-form-urlencoded
form-data multipart/form-data

Request URLs


A GET or POST request can be made to the hosted version of Num-Verify 

GET : https://num-verify.herokuapp.com/api/fetch/all

POST : https://num-verify.herokuapp.com/api/fetch/

Using cURL


GET 

$ curl -o result.txt  https://num-verify.herokuapp.com/api/fetch/all/

POST 

$ curl -d "NUM=8908989089&NUM_PRE=91" -H "Content-Type:application/x-www-form-urlencoded" https://num-verify.herokuapp.com/api/fetch/

JSON Response


{
"EXISTS":true,
"VERIFIED":true,
"CTRY":"India",
"PREFIX":91,
"ALPHA_1":"IN",
"ALPHA_2":"IND",
"NUM_CODE":"356",
"msg":"Number verified successfully"
}

KEY VALUE IMPLICATION
EXISTS true The sent prefix is present in the database , but the number may or may not match with the prefix.
VERIFIED true The sent phone number has successfully matched with the prefix.
CTRY COUNTRY_NAME Name of the Country , the phone number belongs to.
msg MESSAGE Status of the request after processing.
